# Neoflam Uzbekistan — Sales Intelligence Dashboard

Внутренний аналитический дашборд филиала Neoflam в Узбекистане. Отображает спрос, конверсию, инсайты клиентов и план действий.

---

## 🚀 Как запустить локально

Просто открой `index.html` в браузере двойным кликом. Никаких билдов, никаких зависимостей. Это статический сайт на чистом HTML/CSS/JS.

---

## 🌐 Как залить на GitHub Pages (бесплатный хостинг)

### Шаг 1. Создай репозиторий на GitHub
1. Зайди на [github.com](https://github.com) → войди в аккаунт
2. Нажми **«+» → New repository**
3. Назови, например, `neoflam-dashboard`
4. Тип — **Public** (для бесплатного GitHub Pages)
5. Нажми **Create repository**

### Шаг 2. Загрузи файлы
1. На странице нового репозитория нажми **uploading an existing file**
2. Перетащи **все файлы из ZIP** (но не сам ZIP — распакуй заранее)
3. Нажми **Commit changes**

Структура должна быть такой:
```
neoflam-dashboard/
├── index.html
├── css/
│   └── style.css
├── js/
│   └── app.js
└── README.md
```

### Шаг 3. Включи GitHub Pages
1. В репозитории зайди в **Settings → Pages**
2. Source → выбери ветку **main**, папка **/ (root)**
3. Нажми **Save**
4. Через 1-2 минуты сайт будет доступен по адресу:
   ```
   https://ТВОЙ_USERNAME.github.io/neoflam-dashboard/
   ```

### Шаг 4 (опционально). Подключить свой домен
В Settings → Pages → Custom domain → введи свой домен (например `dashboard.neoflam.uz`).

## 🔄 Как обновить данные

Все данные сейчас «зашиты» прямо в HTML. Чтобы обновить цифры:

1. Открой `index.html` в любом редакторе (VS Code, Notepad++, даже Блокнот)
2. Найди нужное число (поиск Ctrl+F)
3. Замени на актуальное
4. Загрузи обновлённый файл на GitHub

**В будущем** (когда сделаем интеграцию с Google Sheets) дашборд будет тянуть данные автоматически — никакого ручного редактирования.

## 📊 Структура дашборда

1. **Hero** — главные KPI крупно
2. **KPI Grid** — 6 карточек с ключевыми метриками
3. **Products** — топ товаров, спрос vs продажи (с графиком)
4. **Funnel** — воронка от запроса до покупки (где утечки)
5. **Insights** — голос клиента, частые слова, инсайты
6. **Actions** — план действий с приоритетами P0/P1/P2
7. **Benchmarks** — сравнение с рынком
8. **Forecast** — три сценария на 30 дней
9. **Data Source** — пайплайн данных
